devDependencies和dependencies区别

devDependencies用于本地环境开发时候
  1. devDependencies是只会在开发环境下依赖的模块,生产环境不会被打入包内。
  2. 安装模块时如果不需要发布到线上环境,使用 –save-dev
dependencies用户发布环境
  1. dependencies依赖的包不仅开发环境能使用,生产环境也能使用。
  2. 安装模块时如果需要发布到线上环境,使用 –save

JavaScript遍历数组方法

JavaScript遍历数组for-of
JavaScript遍历数组forEach
  1. forEach没有返回值
  • 参数1: 每一项的值
  • 参数2: 索引
JavaScript遍历数组map
  1. map有返回值(返回数组)
  • 参数1: 每一项的值
  • 参数2: 索引
JavaScript遍历数组filter
  1. filter有返回值(返回数组),返回的是满足条件的数组
  • 参数1: 每一项的值
  • 参数2: 索引
JavaScript遍历数组every
  1. every有返回值(返回boolean),遍历数组中所有的值,如果全都满足条件返回true,只要有一个没有满足条件,返回false
  • 参数1: 每一项的值
  • 参数2: 索引
JavaScript遍历数组some
  1. some有返回值(返回boolean),如果有一个满足条件返回true
  • 参数1: 每一项的值
  • 参数2: 索引
JavaScript遍历数组reduce
  1. reduce有返回值(返回累加值,如果是number类型的就是计算结果,字符串就是拼接后的字符串),累加数组中的值
  • 参数1: 总计
  • 参数2: 当前值

JavaScript基础

JavaScript基本类型和引用类型
  1. 基本类型: string number blooean null undefined
  2. 复杂类型/引用类型: arr obj function
  3. 基本类型:存放在栈中
  4. 复杂类型/引用类型:引用地址存放在栈里面,数据放在堆里面
JavaScript判断类型typeof
JavaScript函数声明和函数表达式
  1. 函数声明调用可以在前,函数表达式必须要先声明才能调用
JavaScript检查属性
  1. in: 用in检查是否该对象有此属性,如果有返回true否则返回false
  2. hasOwnProperty: 用hasOwnProperty判断对象中是否有该属性,返回blooean 总结: in和hasOwnProperty两者的区别: in会先从原型中去找是否有该属性再找自身是否有此属性,hasOwnProperty只会找自身不会找原型中的属性
